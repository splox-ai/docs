---
title: 'Create Chat'
openapi: 'POST /chats'
---

Creates a new chat session. A chat is required before running a workflow via the API â€” the chat ID is passed to `POST /workflow-requests/run`.

## Usage

```bash
curl -X POST https://app.splox.io/api/v1/chats \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My Chat Session",
    "resource_type": "workflow",
    "resource_id": "0199e001-a23b-7c8d-1234-567890abcdef"
  }'
```

## Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Display name for the chat session |
| `resource_type` | string | Yes | Type of resource: `workflow`, `gui`, `api`, or `agent` |
| `resource_id` | string | Yes | ID of the resource (workflow ID, agent ID, etc.) |
| `metadata` | object | No | Optional metadata |

## Response

```json
{
  "id": "0199f200-a11b-7c8d-4444-888890abcdef",
  "name": "My Chat Session",
  "user_id": "0199d001-c45d-9e0f-3456-789012cdef01",
  "resource_type": "workflow",
  "resource_id": "0199e001-a23b-7c8d-1234-567890abcdef",
  "is_public": false,
  "created_at": "2025-10-22T10:00:00Z",
  "updated_at": "2025-10-22T10:00:00Z"
}
```

## Resource Types

| Type | Description |
|------|-------------|
| `workflow` | Chat associated with a specific workflow |
| `gui` | Chat from the desktop/GUI application |
| `api` | Chat created via API integration |
| `agent` | Chat with a published agent |

## Notes

<Info>
**Authentication required:** The chat is automatically associated with the authenticated user.
</Info>
