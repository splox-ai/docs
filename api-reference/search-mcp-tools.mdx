---
title: 'Search MCP Tools'
openapi: 'POST /mcp-tools/search'
---

Searches MCP servers and returns connection status metadata similar to tool_router behavior.

## Usage

<CodeGroup>

```bash cURL
curl -X POST https://app.splox.io/api/v1/mcp-tools/search \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "search_query": "slack",
    "limit": 10,
    "offset": 0
  }'
```

```python Python
from splox import SploxClient

client = SploxClient(api_key="YOUR_API_KEY")
resp = client.mcp.search(search_query="slack", limit=10, offset=0)
for item in resp.results:
    print(item.name, item.is_user_connected)
```

```typescript Node.js
import Splox from "splox";

const client = new Splox("YOUR_API_KEY");
const resp = await client.mcp.search({
  search_query: "slack",
  limit: 10,
  offset: 0,
});
for (const item of resp.results) {
  console.log(item.name, item.is_user_connected);
}
```

```go Go
resp, err := client.MCP.Search(ctx, &splox.SearchParams{
    SearchQuery: "slack",
    Limit:       10,
    Offset:      0,
})
if err != nil {
    panic(err)
}
for _, item := range resp.Results {
    fmt.Println(item.Name, item.IsUserConnected)
}
```

</CodeGroup>

## Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `search_query` | string | No | Search text for MCP name/description/url. |
| `limit` | integer | No | Max catalog results to return (default 10, max 100). |
| `offset` | integer | No | Offset for paginated search results. |

## Response

```json
{
  "results": [
    {
      "name": "Slack",
      "description": "Slack MCP server",
      "is_user_connected": false,
      "connect_link": "https://app.splox.io/tools/connect?server_id=...",
      "mcp_url": "https://mcp.slack.com"
    }
  ],
  "total": 1,
  "limit": 10,
  "offset": 0
}
```
