## Memory Edges

**Purpose:** Connect memory nodes to LLM nodes for conversation history

Memory edges are special bottom-facing connections that link memory nodes to LLM nodes, enabling stateful conversations with persistent chat history.

**In this example:**
- The **LLM Node** has a **MEMORY** input handle (bottom center)
- A **memory edge** (dashed line with database icon) connects from the **Chat Memory Node**
- The **Chat Memory Node** shows:
  - Context Window Size configuration
  - Memory Load status (Empty)
  - Token count (0 tokens)
- This provides conversation history to the LLM across multiple turns

**Characteristics:**
- **Bottom Connection:** Memory edges connect to the bottom handle of LLM nodes
- **Special Styling:** Dashed line with database icon
- **One-to-One:** Each LLM can have only one memory node
- **Optional:** Not all LLMs need memory

**How It Works:**

1. **History Loading:** When the LLM executes, it loads chat history from the memory node
2. **Context Injection:** Messages are prepended to the LLM's input
3. **Trimming:** Old messages are removed based on memory limits
4. **Summarization:** Optionally, trimmed messages are summarized
5. **New Messages:** LLM responses and tool calls are saved to memory

**Use Cases:**
- Multi-turn conversations
- Customer support chat
- AI assistants with context
- Personalized interactions

<Info>
  Memory edges use the `is_memory` flag and connect to a special handle on LLM nodes.
</Info>
